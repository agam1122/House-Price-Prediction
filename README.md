# Project-Gurgaon-VS_code

## Overview

This project is a practical machine learning pipeline built with Python and `scikit-learn` for predicting housing prices. It demonstrates data preprocessing, training, serialization, and prediction steps using a housing dataset.

The project includes:
- a feature engineering and preprocessing pipeline
- a `RandomForestRegressor` model
- model and pipeline serialization via `joblib`
- prediction output generation to CSV

## Project Structure

- `main.py` - Main script that trains the model if not already saved, then loads the saved pipeline and model to make predictions on input data.
- `housing.csv` - Original housing dataset used for training and sampling.
- `input_data.csv` - Generated input dataset used for prediction after a stratified train/test split.
- `output.csv` - Generated file containing predicted `median_house_value` values.
- `model.pkl` - Serialized trained model.
- `pipeline.pkl` - Serialized preprocessing pipeline.
- `.gitignore` - Folder-level ignore rules for generated files and local settings.
- `.vscode/settings.json` - VS Code settings for this folder.

## Key Features

- Stratified train/test split based on `median_income` categories
- Numerical imputation with median values
- Standard scaling of numerical features
- Categorical imputation and one-hot encoding
- Reusable preprocessing pipeline with `ColumnTransformer`
- Model persistence using `joblib`
- Automatic inference and output generation to CSV

## Requirements

- Python 3.8+ (recommended)
- `numpy`
- `pandas`
- `scikit-learn`
- `joblib`

## Usage

Run the main script from the project folder:

```bash
python main.py
```

### What happens when you run it

- If `model.pkl` does not exist:
  - Load `housing.csv`
  - Add an `income_cat` label based on `median_income`
  - Perform a stratified split and create `input_data.csv`
  - Train a `RandomForestRegressor`
  - Save the trained model and preprocessing pipeline
- Always:
  - Load `input_data.csv`
  - Transform it using the saved pipeline
  - Predict `median_house_value`
  - Write predictions to `output.csv`

## Workflow Details

### Preprocessing pipeline

The pipeline handles both numerical and categorical columns:
- Numerical features:
  - `SimpleImputer(strategy='median')`
  - `StandardScaler()`
- Categorical features:
  - `SimpleImputer(strategy='most_frequent')`
  - `OneHotEncoder(handle_unknown='ignore')`

### Model training

- Uses `RandomForestRegressor(random_state=42)`
- Fits on preprocessed training data
- Saves the model to `model.pkl`

## Notes

- `input_data.csv` and `output.csv` are generated by the script and should not be tracked in Git.
- The `.gitignore` file excludes local artifacts like `*.pkl`, generated CSV files, `__pycache__`, and VS Code settings.
